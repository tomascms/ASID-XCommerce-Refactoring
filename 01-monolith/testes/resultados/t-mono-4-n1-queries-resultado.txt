=== T-MONO-4: N+1 Queries — Checkout com 4 produtos / 5 itens ===
Data: 2026-05-27
Carrinho: produto 0 (qty 2), produto 1 (qty 1), produto 2 (qty 1), produto 3 (qty 1)

QUERIES EMITIDAS PELO HIBERNATE DURANTE GET /rest/order/checkout:

 1. SELECT product+brand+category WHERE product.id=0   ← produto 0
 2. SELECT category (parent_category lazy load)        ← categoria do produto 0
 3. SELECT product+brand+category WHERE product.id=1   ← produto 1
 4. SELECT category (parent_category lazy load)        ← categoria do produto 1
 5. SELECT product+brand+category WHERE product.id=2   ← produto 2
 6. SELECT category (parent_category lazy load)        ← categoria do produto 2
 7. SELECT product+brand+category WHERE product.id=3   ← produto 3
 8. SELECT category (parent_category lazy load)        ← categoria do produto 3
 9. SELECT user WHERE user.id=1                        ← validação do utilizador
10. SELECT nextval('hibernate_sequence')               ← geração do ID da order
11. INSERT INTO "order" (...)                          ← criação da order

TOTAL: 11 queries para 4 produtos distintos / 5 itens

PADRÃO N+1:
  N = número de produtos distintos no carrinho
  Queries = 1 (user) + N (product) + N (category parent) + 1 (sequence) + 1 (insert order)
  Com N=4: 1 + 4 + 4 + 1 + 1 = 11 queries

NOTA PARA RELATÓRIO (§2.2 / Fig. 3):
  Uma query optimizada poderia obter tudo com 1 JOIN:
  SELECT o, ol, p, b, c FROM Order o JOIN o.orderLines ol JOIN ol.product p
  JOIN p.brand b JOIN p.category c WHERE o.user.id = ?
  → 1 query em vez de 11.
